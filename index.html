Tech Challenge Kick-off

2017-01-15

At some point around the holidays I read a summary of someone’s “52 week technology challenge.”

The premise was that they picked a new technology every week and learned enough of it to produce a small project.

I’ve decided to borrow this idea as the inital basis for a blog and also as a means to cross off some things off of my “I should try that” list that’s been growing for quite some time.

However, between work and play, learning producing a project that’s beyond trivial every week on a new technology seems like I’m setting myself up for failure, so I’m going to shoot to do one every two weeks instead.

I’m also going to cheat a little and call this blog project #1. Hexo is completely new to me and my markdown skills were, and still are, probably analagous to fingerpainting.

This one will be more of an ongoing project as I’ll hopefully add more than just status updates to the blog throughout the year and there’s plenty of customization work to do as of today.

Blog To Do List:

  • Publish to github pages
  • Remove Chinese tracker and comment system from theme
  • Pick a name
  • Add Disqus
  • Update logo
  • Add some custom background photos
  • Get About section working
  • Get Tags/Categories sections working
  • Get theme changes into existing repo or fork the original

Next Project: NPM

In tandem with making these changes, I’ll be starting on project #2.
The technology here is npm packaging; the project is building an e2e test page object generator for Angular using typescript.
The generator will be consumable in any npm toolchain and published to the npm package repository.
